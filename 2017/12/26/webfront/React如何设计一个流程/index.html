React如何设计一个流程

发表于 2017-12-26 | 分类于 web前端 |

流程在前端应用（App）上的使用是非常广的，比如注册、申请功能、下单购买、开票、使用引导等。React 组件实际上是一个状态机，所以，我们可以基于这个特点来设计一个简单的流程。

假设我们现在要设计一个申请服务的流程。

结构

我们先来看一下结构图。其中包含几个界面之间的流转逻辑。

一个流程中按用途来分可以包含两种组件，容器组件和 UI 组件。

实现

容器组件

控制流程流转的流程组件我们称之为容器组件（Container），在实际界面中没有显式的存在，主要作用是控制流程的状态和流程的流转，传递数据流等。

状态

在容器组件中，设置一个 state 作为状态机的状态，step。step 的初始状态为0。

每一个状态对应一个UI组件的渲染函数。
